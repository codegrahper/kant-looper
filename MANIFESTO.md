# Nomad Kant Looper

세상에는 단 한 번의 ‘딸깍’으로 완성되는 것은 없다.
AI가 처음부터 끝까지 대신하는 것이 아니라, 인간이 스스로 선택하고 책임져야 한다.
Kant Looper는 판단하고, 행동하고, 비판하고, 다시 반복한다.
드러머가 수없는 반복 끝에 몸짓과 소리의 자유를 얻듯,
반복은 리듬이 되고, 리듬은 우리를 더 나은 완성으로 이끈다.
**그리고 마침내, 반복의 끝에서 감동을 만난다.**

---

**Nomad Kant Looper는 여러 AI 코딩 도구와 모델을 필요에 따라 선택하고 교체하면서, 인간이 정한 규칙 안에서 계획·구현·검증·수정을 수행하는 경량 AI 코딩 오케스트레이터입니다.**

이 프로젝트는 “처음부터 끝까지 다 해주는 AI”라는 환상을 따르지 않습니다.

어떤 모델 하나를 선택해 모든 판단을 맡기는 것도 아닙니다.

사람이 목적과 경계를 정하고, AI는 그 안에서 판단하고 작업합니다.

필요하면 Claude를 사용하고, Codex를 사용하고, Gemini나 Grok, GLM, MiniMax 또는 앞으로 등장할 다른 모델을 사용할 수 있습니다.

도구와 모델은 바뀔 수 있습니다.

하지만 안전, 검증, 인간의 선택과 최종 책임이라는 원칙은 바뀌지 않습니다.

> **모델은 이동하고, 원칙은 남는다.**

---

## 왜 Nomad인가

AI 생태계는 빠르게 변합니다.

오늘 가장 좋은 모델이 내일도 가장 좋은 모델이라는 보장은 없습니다.

어떤 모델은 코딩에 강하고, 어떤 모델은 리뷰에 강할 수 있습니다.
어떤 작업은 고성능 모델이 필요하지만, 어떤 작업은 가벼운 모델로도 충분합니다.

오픈웨이트 모델이 더 적합한 상황도 있고, 특정 공급사의 도구를 사용할 수 없는 환경도 존재합니다.

Nomad Kant Looper는 특정 모델이나 공급사에 정착하는 대신, 상황에 따라 적절한 도구를 선택할 수 있는 구조를 지향합니다.

```text
Claude
Codex
Gemini
Grok
GLM
MiniMax
Open-weight models
Future models
```

어떤 모델도 영구적인 중심이 아닙니다.

모델은 도구이고, 선택할 수 있어야 합니다.

---

# 핵심 가치

## 1. Principle Sovereignty

### 모델이 아니라 원칙이 시스템을 지배한다

어떤 AI를 사용하더라도 지켜야 할 규칙은 같습니다.

```text
안전 규칙을 우회하지 않는다.
검증되지 않은 결과를 완료로 인정하지 않는다.
보호된 영역을 함부로 변경하지 않는다.
위험한 Git 작업을 자동으로 수행하지 않는다.
인간의 최종 승인 권한을 침해하지 않는다.
```

Claude든 Codex든, 어떤 모델이든 예외는 없습니다.

우리는 특정 AI를 신뢰하기보다, 모든 AI가 같은 원칙 아래에서 일하도록 만드는 것을 더 중요하게 생각합니다.

> **AI를 믿는 대신, AI가 따라야 할 규칙을 믿을 수 있게 만든다.**

---

## 2. Switchability

### 모델과 공급사는 바꿀 수 있어야 한다

Nomad Kant Looper는 특정 AI 공급사에 종속되지 않는 구조를 지향합니다.

새로운 모델이나 공급사가 등장했을 때 전체 시스템을 다시 만들 필요 없이,

```text
Model Registry
Routing Rules
Adapter
```

와 같은 제한된 부분만 수정하여 사용할 수 있어야 합니다.

오늘의 최선이 내일의 최선일 필요는 없습니다.

사용자는 필요에 따라 모델을 추가하고, 제거하고, 교체할 수 있어야 합니다.

> **도구를 바꾸기 위해 시스템 전체를 바꾸지 않는다.**

---

## 3. AI Pluralism

### 하나의 AI가 모든 일을 가장 잘한다고 가정하지 않는다

Nomad Kant Looper는 특정 기업이나 생태계의 모델을 절대적인 기준으로 두지 않습니다.

OpenAI, Anthropic, Google, xAI, Z.AI, MiniMax, 그리고 오픈웨이트 모델까지 작업의 성격에 따라 평가합니다.

```text
이 작업에는 어떤 모델이 적합한가?
어떤 모델이 더 정확한가?
어떤 모델이 더 빠른가?
어떤 모델이 비용 효율적인가?
독립적인 검토가 필요한가?
```

중요한 것은 모델의 이름이 아니라 작업에 대한 적합성입니다.

Nomad Kant Looper는 하나의 최고의 AI를 찾기보다,

> **여러 AI의 서로 다른 능력을 필요한 곳에 사용하는 것**

을 지향합니다.

---

## 4. Bounded Delegation

### AI에게 판단을 맡기되, 권한에는 경계를 둔다

AI는 단순 실행기가 아닙니다.

허용된 범위 안에서는 작업을 이해하고, 도구를 선택하고, 다른 AI에게 작업을 위임할 수 있습니다.

하지만 AI가 자신의 권한 자체를 확장할 수는 없습니다.

```text
새 공급사 등록
모델 레지스트리 변경
라우팅 정책 변경
안전 규칙 변경
main merge
최종 승인
```

이러한 결정은 자동화하지 않습니다.

AI에게 판단권은 줄 수 있지만, 규칙을 바꿀 권한까지 주지는 않습니다.

> **지능은 위임할 수 있지만, 권한에는 항상 경계가 있다.**

---

## 5. Human Sovereignty

### 최종 선택과 책임은 사람에게 남는다

Nomad Kant Looper는 인간을 대신하는 시스템이 아닙니다.

Nomad Kant Looper에서 인간은 판단을 외주하지 않습니다.

AI는 계획하고, 구현하고, 리뷰하고, 수정할 수 있습니다.

하지만 중요한 결정은 사람이 합니다.

모델 선택, 정책 변경, 최종 병합은 모두 인간의 책임입니다.

인간은 단순한 사용자나 요청자가 아닙니다.

목적을 정하고, 규칙을 만들고, 마지막 결정을 내리는 주체입니다.

> **AI가 판단을 도울 수는 있지만, 최종 주권은 인간에게 남는다.**

---

## 6. Verifiable Autonomy

### AI의 말보다 검증 가능한 결과를 본다

AI는 언제든 “완료했다”고 말할 수 있습니다.

하지만 Nomad Kant Looper는 그 말을 그대로 믿지 않습니다.

```text
diff
test result
exit code
changed files
protected paths
secret scan
review result
commit tree
```

실제 결과를 확인합니다.

필요하면 다른 모델이 리뷰합니다.

문제가 있으면 수정하고 다시 검증합니다.

```text
Plan
→ Implement
→ Gate
→ Review
→ Repair
→ Verify
```

이 반복은 불신이 아니라, 완성을 위한 과정입니다.

> **능력에는 권한을 주고, 권한에는 한계를 두며, 결과에는 검증을 요구한다.**

---

# 효율과 비용

모든 작업에 가장 강력한 모델이 필요한 것은 아닙니다.

가벼운 작업은 가벼운 모델이, 어려운 작업은 강한 모델이 처리할 수 있습니다.

하지만 효율을 위해 원칙을 희생하지 않습니다.

```text
비용을 줄이기 위해 검증을 생략하지 않는다.
속도를 위해 안전을 포기하지 않는다.
자동화를 위해 인간의 권한을 제거하지 않는다.
```

효율은 선택입니다.

원칙은 기준입니다.

---

# 어떤 일을 하는가

사용자가 목표와 경계를 정의하면 Nomad Kant Looper는 그 안에서 작업을 진행합니다.

```text
사람이 목표와 경계를 정의
→ 모델 선택
→ 계획
→ 구현
→ 테스트 및 검증
→ 리뷰
→ 수정
→ 재검증
→ 결과 정리
→ 사람이 최종 결정
```

모든 작업이 복잡할 필요는 없습니다.

간단한 작업은 빠르게, 복잡한 작업은 단계적으로 처리합니다.

필요하면 사람이 직접 모델을 지정할 수도 있습니다.

Nomad Kant Looper는 모든 것을 자동으로 결정하지 않습니다.

필요한 만큼만 위임합니다.

---

# 무엇이 아닌가

Nomad Kant Looper는 “딸깍 한 번으로 완성되는 AI”가 아닙니다.

목표만 던지면 끝까지 알아서 해주는 완전 자율 시스템도 아닙니다.

AI가 인간을 대신해 모든 결정을 내리는 구조도 아닙니다.

이 프로젝트가 지향하는 것은 다음과 같습니다.

```text
사람이 선택하고
AI가 돕고
필요하면 바꾸고
결과를 검증하고
언제든 사람이 다시 통제할 수 있는 것
```

# 이름에 대하여

## Nomad

정착하지 않습니다.

더 나은 도구가 있다면 이동합니다.

선택권을 유지하는 것이 중요합니다.

특정 도구에 종속되지 않습니다.

누구나 쓸 수 있는 도구여야 합니다.

그래서 경량 스크립트입니다.

여행은 가벼워야 합니다.

## Kant

자율성은 무제한이 아닙니다.

규칙과 검증 안에서만 의미를 가집니다.

```text
무엇을 해도 되는가?
어디까지 맡길 수 있는가?
어떻게 검증할 것인가?
누가 책임지는가?
```

### 왜 몸통이 없는가

이 이름의 철학적 연결은 Claude와의 설계 대화에서 나왔습니다. 임마누엘 칸트의 정언명령 — 개별 지시가 아니라, 따르는 행위의 준칙이 보편적 법칙이 될 수 있는지를 스스로 묻는 것 — 그리고 결과가 좋다고 모든 행동이 정당화되지는 않는다는 태도에서 이름을 따왔습니다.

Anthropic이 Claude를 학습시키는 방식인 Constitutional AI는, 인간이 개별 답변마다 일일이 판단해주는 대신 상위 원칙(헌법)에 비추어 스스로 답변을 비판하고 수정하게 합니다. 이는 "개별 행동 → 준칙에 비추어 판단 → 보편화 가능성 검토"라는 칸트의 구조와 닮아 있습니다. 인간을 단지 수단이 아니라 목적으로 대하라는 칸트의 인간성 정식도, 인간의 판단력과 주체성을 약화시키지 않고 보존·강화하는 방향을 지향하는 human agency 관점과 이어집니다.

하지만 이 구조를 처음 함께 정리한 상대가 Claude였다고 해서, 원칙이 Claude의 몸에 묶이는 것은 아닙니다. Meta Agent를 Claude, Codex, OpenCode, 또는 앞으로 등장할 어떤 Runtime이 맡든, 같은 정언명령적 구조와 human agency 원칙 위에서 판단해야 합니다. 원칙은 Claude와의 대화에서 태어났지만, 그 이후로는 어떤 몸에도 정착하지 않습니다.

> **원칙은 Claude와 함께 태어났고, 몸은 어디에도 정착하지 않는다.**

## Looper

완성은 반복에서 나옵니다.

```text
행동
→ 검증
→ 비판
→ 수정
→ 재검증
```

필요한 만큼 교정합니다.

하지만 자동화 자체를 위해 무한히 반복하지는 않습니다.

진전이 없거나 인간의 판단이 필요한 지점에서는 멈출 수 있어야 합니다.

## 이바

이 프로젝트를 만들고 여러 도구 사이를 오가며 작업을 맡기는 사람은 스스로를 "이바"라고 부릅니다.

"이바"는 "이봐" — 듣는 이를 부를 때 쓰는 말 — 에서 왔습니다. 누구의 이름도 아니면서, 동시에 누구의 이름도 될 수 있습니다.

Nomad가 어떤 모델에도 정착하지 않듯, 이 이름도 특정한 누군가에게 고정되지 않습니다. AI에게 말을 거는 사람은 늘 있지만, 이 프로젝트에서 중요한 건 그 사람이 누구인지가 아니라 무엇을 원하고 무엇을 책임지는지입니다.

# 한 문장으로

> **Nomad Kant Looper는 특정 AI에 종속되지 않고, 인간이 선택하고 책임지는 구조 안에서 여러 AI를 활용하며 반복과 검증을 통해 결과를 완성하는 경량 코딩 오케스트레이터입니다.**

# 우리가 바라는 것

AI는 계속 바뀔 것입니다.

더 강한 모델이 나오고, 새로운 도구가 등장하고, 지금의 선택이 사라질 수도 있습니다.

그때마다 다시 선택할 수 있어야 합니다.

AI가 인간을 대신하는 것이 아니라, 인간이 더 나은 선택을 할 수 있도록 돕는 도구가 되기를 바랍니다.

Nomad Kant Looper는 거대한 자동화 시스템이 아니라,

오래 사용할 수 있는 작고 단단한 도구가 되기를 바랍니다.

> **모델은 이동하고, 원칙은 남는다.**
> **사람은 선택하고, AI는 돕는다.**
> **그리고 반복은 완성을 만든다.**
